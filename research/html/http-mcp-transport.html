Why HTTP MCP?

stdio works great for local tools — the AI tool spawns PonderDB as a child process. But some scenarios need HTTP:

  • Remote access — MCP server on cloud VM, clients connect over network
  • ChatGPT / web-based tools — Can’t spawn child processes
  • Shared server — Multiple users/tools connecting to one instance
  • Containerized deployments — Docker, Kubernetes
  • Cross-machine memory — Access from laptop, desktop, CI/CD

Architecture

Client (AI Tool) PonderDB Server (Hono) │ │ │ POST /mcp │ │ { "method": "initialize" } →→→→│ │ │ Create transport + MCP server │ ←←← SSE: initialize result │ Store session in map │ + mcp-session-id header │ │ │ │ POST /mcp │ │ mcp-session-id: <uuid> │ │ { "method": "tools/call", │ │ "name": "remember" } →→→→│ Route to stored transport │ │ │ ←←← SSE: tool result │ │ │ │ DELETE /mcp │ │ mcp-session-id: <uuid> →→→→│ Clean up session

Session Management

// Map of session ID -> transport
const sessions = new Map<string, WebStandardStreamableHTTPServerTransport>();

// New session: transport created with UUID generator
const transport = new WebStandardStreamableHTTPServerTransport({
  sessionIdGenerator: () => randomUUID(),
  onsessioninitialized: (id) => sessions.set(id, transport),
  onsessionclosed: (id) => sessions.delete(id),
});

// Each session gets its own MCP server
const mcpServer = createMcpServer(store, embedder);
await mcpServer.connect(transport);

Hono Integration

The SDK’s WebStandardStreamableHTTPServerTransport uses Web Standard Request/Response — perfect fit for Hono. c.req.raw passes directly, zero adapter overhead.

router.all("/", async (c) => {
  return transport.handleRequest(c.req.raw);
});

Security

Current: No Auth on /mcp

MCP HTTP endpoint currently has no API key auth. Local-only binding (127.0.0.1) — not exposed to network by default.

Generic MCP Client

import { StreamableHTTPClientTransport } from "@modelcontextprotocol/sdk/client/streamableHttp.js";
import { Client } from "@modelcontextprotocol/sdk/client/index.js";

const client = new Client({ name: "my-app", version: "1.0" });
const transport = new StreamableHTTPClientTransport(
  new URL("http://127.0.0.1:7437/mcp")
);
await client.connect(transport);

const result = await client.callTool("remember", {
  key: "test/hello",
  content: "Hello from HTTP MCP"
});
