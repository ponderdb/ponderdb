Data Flow

Memory Write Flow

1. Receive

API Gateway → Rate limit → Auth check

2. Validate

Input validation → Generate key → Auto-categorize

3. Store

Write to PostgreSQL (source of truth)

4. Async

Queue: embedding gen → Vector DB sync

5. Notify

Invalidate cache → WebSocket event → Audit log

Memory Search Flow

1. Embed Query

Generate query embedding (local or API)

2. Parallel Search

Qdrant (vector) + PostgreSQL (full-text) + metadata filter

3. Merge & Rank

Reciprocal Rank Fusion + recency boost + access frequency

4. Return

Top-K results with relevance scores

Database Schema (Core Tables)

-- Memories (partitioned by user_id hash)
memories: id, user_id, team_id, key, content, content_ts (tsvector),
          category_id, tags[], metadata (jsonb), source, byte_size,
          version, confidence, access_count, last_accessed, expires_at,
          created_at, updated_at
          UNIQUE(user_id, key)

-- Users + Auth

-- Teams

Hybrid Search Algorithm

1. Generate query embedding (768-dim)
2. Parallel:
   a. Qdrant: top-50 by cosine similarity (with payload filters)
   b. PostgreSQL: top-50 by ts_rank (full-text search)
3. Normalize scores (both to 0-1)
4. Reciprocal Rank Fusion: score = sum(1 / (60 + rank_i))
5. Recency boost: score *= 1 + (0.1 * recency_factor)
6. Access boost: score *= 1 + (0.05 * log(access_count + 1))
7. Sort by final score, return top-K

Caching Strategy

Layer 1: CDN (Cloudflare)       — Static assets, immutable content by version hash
Layer 2: Redis/Valkey           — Hot memories (1hr TTL), search cache (5min), sessions
Layer 3: Application LRU        — In-process cache (100 items), model cache, config

Cache invalidation: Write-through + Redis Pub/Sub for cross-instance eviction
